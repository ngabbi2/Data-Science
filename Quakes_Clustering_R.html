The CH index is the ratio of within cluster distance to between cluster distance. Ideally for tighter and clearly separated clusters the value of CH index has to be highest.

For WHS dataset the ch index value is highest at 2 clusters and then it reduces consistently indicating that there are no really good clusters in the dataset.

For Quakes data set the value of Ch index is highest at 4 and 5 clusters after that the Ch index value reduces gradually. Indicating that the number of clusters in the quakes data set is either 4 or 5.

Gap statistics

gapstat <- function(m, k, N=20){
w=numeric(N)
x=numeric(20)

for (i in 1:N)
{
  m.new = apply(scale(m),2,function(x){
    runif(length(x),min=min(x),max=max(x))
  })
  kf = kmeans(m.new,k)
  w[i] = kf$tot.withinss
}
return(list(LW=mean(log(w)),SE=sd(log(w))/sqrt(N)))
}

calcgap <- function(dataset){
  
gap = numeric(20)
se = numeric(20)
for(k in 1:20){
  kf = kmeans(scale(dataset),k)
  sim = gapstat(scale(dataset),k)
  gap[k]=sim$LW-log(kf$tot.withinss)
  se[k]=sim$SE
}
plot(1:20,gap,pch=19,type="b")

}

calcgap(whsAnnBdatNum)

calcgap(quakes)

The gap statistic for K means clustering on scaled WHS data keeps increasing and it does not suggest any presence of clusters in the dataset. where as for the quakes dataset the gap statistic increases upto 5 clusters and decreases for 6 clusters. Indicating first peak at 5 clusters. This can be used to deduce that there is cluster presence in the quakes data set with 5 clusters.

Between/within variance in hierarchical clusters

within = function(d, clust){
  w = numeric(length(unique(clust)))
  for(i in sort(unique(clust))){
    members = d[clust ==i, , drop=F]
    centroid = apply(members,2,mean)
    members.diff = sweep(members,2,centroid)
    w[i]= sum(members.diff^2)
  }
  return(w)
}

between = function(d,clust){
  b=0
  total.mean=apply(d,2,mean)
  for(i in sort(unique(clust))){
    members=d[clust==i,,drop=F]
    centroid=apply(members,2,mean)
    b=b+nrow(members)*sum((centroid-total.mean)^2)
  }
  return(b)
}

between_within <- function(dataset){
  
dd = dist(scale(dataset))
hw = hclust(dd,method="ward.D2")
w.tot = numeric(19)
btw = numeric(19)
for(k in 2:20){
  clust = cutree(hw,k=k)
  w = within(scale(dataset),clust)
  w.tot[k-1]=sum(w)
  btw[k-1]=between(scale(dataset),clust)
}
plot(2:20,w.tot,pch=19,type="b")
plot(2:20,btw,pch=19,type="b")
plot(2:20,(btw/(1:19))/(w.tot/(nrow(scale(dataset))-2:20)),pch=19,type="b")
}

Which is not good indication of identifying clusters.

Brute force randomization in hierarchical clustering

Compare distribution of the heights of the clusters defined by hclust with Ward’s clustering of Euclidean distance between countries in scaled WHS dataset and those obtained by applying the same approach to the distances calculated on randomly permuted WHS dataset as illustrated in the lecture slides. Discuss whether results of such brute force randomization are supportive of presence of unusually close or distant sets of observations within WHS data.

brute_force <- function(dataset){
  
dd = dist(scale(dataset))
hw = hclust(dd,method="ward.D2")

ori.heights = hw$height
rnd.heights = numeric()
for(i.sim in 1:100){
  data.rnd <- apply(scale(dataset),2,sample)
  hw.rnd = hclust(dist(data.rnd),method="ward.D2")
  rnd.heights <- c(rnd.heights,hw.rnd$height)
}
plot(ori.heights,rank(ori.heights)/length(ori.heights),col="red",xlab="height",ylab="F(height)",pch=19)
points(rnd.heights,rank(rnd.heights)/length(rnd.heights),col="blue")

}
brute_force(whsAnnBdatNum)   

brute_force(quakes)

The shape of the curve in scaled WHS data set and randomly permutted WHS dataset are similar in shape but there is a differene in values there by indicating the distant set of observations within WHS data set. The quakes data set curve overlaps with scaled and randomly permutted data set there by indicating unsually close set of observations.
